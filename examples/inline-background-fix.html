Inline Background Fix — 验证页

点击 "Run html2canvas" 后，下方 Canvas 输出应与上方 HTML 原始渲染一致：
背景色/背景图应只覆盖每行文字，而不是跨行的整个大矩形。

① 多行 inline — 背景色（黄色应只覆盖文字行，中间空白处不应有色块）
这是一段带有背景色的 inline 文本，它足够长，会在容器中自动换行，从而跨越两行或更多行。修复前背景色会填满整个外接矩形，修复后只覆盖每行文字。
② 多行 inline — 背景渐变（渐变应随每行重复/裁剪，不是一整块）
这是一段带有渐变背景的 inline 文本，同样需要换行才能触发 bug。渐变背景在换行时应该和浏览器原生渲染一致，而不是覆盖整个外接矩形区域。
③ 单行 inline — 背景色（对照组，行为不应改变）
单行文字，背景色正常。
④ 多行 inline — 背景色 + 实线边框（左边框只在首行，右边框只在末行，上下边框每行都有）
这是一段带有背景色和边框的 inline 文本，它会在容器中换行。修复前边框是整个外接矩形的边框，修复后边框应沿每个行框绘制：上下边框每行都有，左边框只在第一行，右边框只在最后一行。
⑤ 多行 inline — 背景色 + 虚线边框
这是一段带有背景色和虚线边框的 inline 文本，同样需要换行才能触发。虚线边框也应该按行框分别渲染，而不是整个外接矩形。
⑥ inline-block — 背景色 + 边框（对照组，行为不应改变）
inline-block 元素背景和边框，不受本次修复影响。
⑦ 多行 inline — 背景色 + border-radius（圆角：首行左侧两角、末行右侧两角；中间行无圆角）
这是一段带有背景色、边框和圆角的 inline 文本，它会在容器中换行。按 CSS 规范，圆角只应出现在元素的起始端（首行左侧）和结束端（末行右侧），中间行框的连接处不应有圆角。
⑧ 单行 inline — 背景色 + border-radius（对照组，四角均应圆角）
单行文字加圆角，四角应全部圆角，与浏览器原生一致。
